NavBuddy

An AI-augmented navigation assistant that uses vision-language models and dashcam footage to deliver context-aware, lane-level driving guidance.

Aron Bakes, Tony Nguyen, Mohammed Elhenawy †, Andry Rakotonirainy †

Read Paper Code
Abstract

Navigation systems such as Google Maps have transformed modern transport, yet their reliance on delayed or ambiguous instructions can increase cognitive load, particularly in complex multi-lane environments and for users with spatial or driving-related anxiety. We present NavBuddy, a zero-shot multimodal prompting framework and open-source toolkit that leverages vision-language models (VLMs) and real-world scene context to transform generic GPS instructions into concise, lane-aware guidance grounded in visible cues. We conduct a preliminary evaluation across frontier and open-source models on a dataset of 100 navigation steps with human-labelled ground truth. Performance is evaluated using semantic similarity (BERTScore F1), lane-change prediction (F1), and next-action accuracy.

NavBuddy-100

100 held-out ground-truth navigation samples from real urban street environments. One dashcam frame per sample (100 images total), paired with overhead OSM maps and turn-by-turn instructions. Human-verified labels for lane-change detection and next-action prediction.

1 — Install
pip install navbuddy
2 — Download NavBuddy-100
navbuddy setup
3 — Run inference
navbuddy evaluate -d ./data/samples.jsonl \
  -m google/gemini-3-flash-preview -n 5

Prompts for your Google Maps API key, downloads 100 frames (~$0.70) with pre-built overhead maps. For 4 frames per step, use --frame-profile sparse4.
